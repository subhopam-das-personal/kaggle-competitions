# Research File R3 — Datasets & Data-Generation Strategy

**Status:** T3 done. **Key reframing:** because the competition's benchmark is a set of six algorithmically-generated task templates, the interesting "dataset" question is *not* "which open corpus should I train on," but rather "**how do I generate and filter CoT traces for the 9,500 provided problems and for auxiliary problems I synthesize myself**." General open reasoning corpora (OpenMathReasoning, OpenR1-Math, NuminaMath, LIMO, s1K, OpenThoughts) are mostly **irrelevant for direct SFT** on this benchmark — they will teach the model a different style of reasoning than what the grader expects.

## 1. The primary dataset: the competition's 9,500 problems

- Provided as part of the Kaggle competition data tab. 6 categories, templated. [S1]
- Ground-truth answers are present, so the competition is **verifier-friendly**: a Python oracle can check every candidate CoT trace for each of the 5 algorithmic categories (Gravitational, Base Conversion, Unit Conversion, Text Encryption, Bit Manipulation) and most Equation-Transformation-Numeric examples. [S2]
- **Do not** directly SFT on the official CoT traces included with the dataset: ~50 % of Bit-Manipulation traces and ~49 % of Equation-Transformation traces are mis-aligned with their boxed answer. [S3]

### Recommended primary data pipeline
1. **Re-solve** each of the 9,500 problems with a deterministic Python oracle per category (straightforward for 5/6 categories).
2. For each oracle solution, **generate a clean CoT trace** by (a) writing a per-category template that expands into a natural-language walkthrough, or (b) prompting a strong open teacher (DeepSeek-R1, gpt-oss-120b, or Gemini 2.0 Flash — all host-cleared for this comp) to produce a trace that *ends* with the oracle-verified `\boxed{...}`.
3. **Reject-sample**: discard any trace whose final boxed answer doesn't match the oracle or whose length exceeds ~6 k tokens (to stay safely below the 7,680 `max_tokens` generation cap).
4. **Mix 75 % reasoning traces + 25 % direct-answer variants** per Unsloth's Nemotron-3 guidance to avoid collapsing the reasoning-mode behavior [S4].
5. Augment Equation-Transformation-Symbolic with **synthetic extra problems** generated by your own template (digit bijection + operator bijection), plus their oracle-verified traces. This is where the score gap lives.

## 3. Licensing red-flags to avoid for prize eligibility

- **Anthropic Claude outputs** in training data → host clearance is soft; Anthropic ToS prohibits "training models that compete with Claude." Low risk for a task-specific rank-32 LoRA, but avoid if aiming for Open Contribution Awards which require public methodology.
- **OpenAI closed models** (GPT-4o, o1, o3, GPT-5) → same language: prohibited for competing-model training. *Open* OpenAI models (gpt-oss-20b, gpt-oss-120b) are **Apache 2.0** and fully fine. [S5]
- **Gemini closed models** → Google ToS forbids "create or improve models similar to a Google Model", but host CPMP explicitly stated this comp is fine since NVIDIA won't reuse participant adapters. [S6]
- **DeepSeek-R1 / R1-0528** → **MIT, fully open for distillation**. Safest teacher. [S5]
- **Qwen3-235B-Thinking / Qwen3-30B-A3B** → open weights, Apache 2.0 — fully fine. [S5]
- For **public notebook publishing (required for all prizes)**, the teacher-model dataset you generate must also be publicly shareable. Use MIT/Apache teachers where possible.

## 4. Recommended data budget for this competition

For a rank-32 LoRA on Nemotron-3-Nano-30B targeting this benchmark, a realistic breakdown for ~9 weeks of work:

| Stage | Target | Data size | Teacher | Cost |
|---|---|---|---|---|
| **Baseline SFT** | Cover 5/6 categories at ≥95 % local | ~3,000 cleaned traces (600/cat × 5) | Python oracles (no LLM) | $0 |
| **Hard-category SFT** | Push Equation-Transformation-Symbolic + Bit-Manipulation | 2,000–5,000 synthetic problems with verified traces | DeepSeek-R1 API (~$0.55/M output tokens) or local Qwen3-30B-Thinking | $10–$50 |
| **Optional GRPO** | Polish decision boundaries | RL environment using Python oracle as reward | (no teacher needed) | $0 in data, GPU-hours only |
| **Non-reasoning mix-in** | 25 % direct-answer traces to preserve chat quality | ~1,000 samples from `nvidia/HelpSteer3` or similar | Already generated | $0 |
| **TOTAL** | | ~6–10 k traces | Mostly oracle + 1 API teacher | **<$100** |

Published precedent for "less is more" regime: **LIMO 817 samples** and **s1 1K samples** both achieved strong reasoning gains on their target benchmark. Expected analogue here: **1–2 k verified traces per category** should saturate the four easy categories; the remaining score lift comes from data quality, not quantity. (Cross-referenced with Unsloth's Nemotron-3 guide which encourages small high-quality sets. [S4])

## 5. Open / unresolved
